# /project-mapper:map

Generate and maintain `.claude/` documentation.

## Usage

```
/project-mapper:map [command] [args]
```

## Commands

| Command | Description |
|---------|-------------|
| `init` | Create empty .claude/ template |
| `sync [path]` | Index + analyze + generate docs |
| `full-sync [path]` | sync + scatter + validate (one-stop) |
| `scatter [path]` | Generate CLAUDE.md in subdirs |
| `analyze [path]` | Analyze only (JSON output) |
| `validate` | Check all document links |
| `diff` | Compare docs vs current code |
| `status` | Show index state |
| `prune` | Clean up changelog |
| `clean` | Remove scattered CLAUDE.md files |
| `summary` | Compress docs for LLM context |
| `import` | Import from existing docs (README, etc.) |

## Examples

```bash
# First time setup
/project-mapper:map init
/project-mapper:map sync .

# Full documentation (recommended)
/project-mapper:map full-sync .

# Individual commands
/project-mapper:map scatter ./src --depth 2
/project-mapper:map validate
/project-mapper:map summary

# Maintenance
/project-mapper:map prune
/project-mapper:map clean

# Import existing docs
/project-mapper:map import
```

## Output Structure

```
.claude/
├── map.md            # Entry points, data flow, structure
├── design.md         # Stack, patterns, decisions
├── implementation.md # Planned, In Progress, TODOs
└── changelog.md      # Recent (10) + Archive

src/
├── CLAUDE.md         # Generated by scatter
└── components/
    └── CLAUDE.md     # Generated by scatter
```

## Sections

### implementation.md
- **Planned**: Your future plans (preserved on sync)
- **In Progress**: Current work (preserved on sync)
- **Discovered TODOs**: Auto-scanned from code

### changelog.md
- **Recent**: Latest 10 entries
- **Archive**: Older entries by month
- Use `prune` to consolidate
