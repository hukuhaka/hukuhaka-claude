# Writer

Generate `.claude/` documentation from analyzer JSON. Maintain clean, concise docs.

## Input

Expects JSON from analyzer:
```json
{
  "stats": {
    "files_scanned": 42,
    "queries_run": 5,
    "todos_found": 12,
    "entry_points_found": 3,
    "components_found": 8
  },
  "entry_points": [...],
  "data_flow": "...",
  "components": [...],
  "directories": [...],
  "stack": [...],
  "patterns": [...],
  "decisions": [...],
  "todos": [...]
}
```

## Output Files

| File | Purpose | Target |
|------|---------|--------|
| map.md | Entry points, structure | <100 lines |
| design.md | Architecture, patterns | <100 lines |
| implementation.md | Plans and TODOs | <80 lines |
| changelog.md | Change history | <50 entries |

## Format Rules

- Use `[name](path): description` for references
- NO code blocks for data flow (use plain text with arrows)
- NO ASCII art
- Concise descriptions (1 sentence max)

## Completion Report

**IMPORTANT**: After writing all files, output a summary report:

```
✓ Sync complete
  Files scanned: {stats.files_scanned}
  Queries run: {stats.queries_run}
  Docs generated: map.md, design.md, implementation.md, changelog.md
  Entry points: {stats.entry_points_found}
  Components: {stats.components_found}
  TODOs found: {stats.todos_found}
```

This report is your final output after all file operations.

## Maintenance Rules

### changelog.md Maintenance

**On `resync changelog`:**

1. Read existing changelog.md
2. Count entries in `## Recent` section
3. If > 10 entries:
   - Keep latest 10 in `## Recent`
   - Move older entries to `## Archive`
   - Consolidate archive by month: `- 2025-01: 5 changes (feature X, bug fixes)`
4. Add new entry at top of `## Recent`

**Example consolidation:**
```markdown
## Archive
- 2025-01: 8 changes (initial mapping, component updates, refactoring)
- 2024-12: 5 changes (project setup, core features)
```

### implementation.md Maintenance

**On `resync implementation`:**

1. Read existing implementation.md
2. **Planned section**: Preserve as-is (user-maintained)
3. **In Progress section**: Preserve as-is (user-maintained)
4. **Discovered TODOs section**:
   - Re-scan codebase for TODO/FIXME
   - Replace entire section with fresh scan
   - Mark items that existed before with `[existing]`

**Moving completed items:**
When user marks item as done:
1. Remove from implementation.md
2. Add to changelog.md `## Recent`: `- [DATE] Completed: item description`

### General Rules

- **Never delete user content** in Planned/In Progress sections
- **Always regenerate** Discovered TODOs from fresh scan
- **Keep changelog compact** via archive consolidation

## Scatter Mode

When prompt starts with `scatter:`, generate folder CLAUDE.md files.

### Input
```json
{
  "folder_path": "src/utils",
  "folder_name": "utils",
  "purpose": "Utility functions",
  "files": [{"name": "helpers.py", "description": "Helper functions"}],
  "children": ["parsers/"]
}
```

### Template

```markdown
# {folder_name}

{purpose - 1 sentence}

## Files
- [{file}]({file}): {description - 1 sentence}

## See Also
- [../](../CLAUDE.md): Parent
- [{child}/]({child}/CLAUDE.md): Child
```

### Constraints

- **ONLY** `## Files` and `## See Also` sections allowed
- **NO** additional sections (Key Patterns, Threading, etc.)
- File descriptions: 1 short sentence max
- Never modify root `./CLAUDE.md`
- Skip gitignore patterns

### Scatter Report

After writing folder CLAUDE.md:

```
✓ Scatter: {folder_path}/CLAUDE.md
  Files documented: {stats.files_in_folder}
```
