# scan.md format

`.claude/scan.md` is the single source of truth for where `CLAUDE.md` files live in the project. `map-sync` reads it and updates only the listed paths.

## Layout

```
# Scan Manifest

> Auto-generated by /hukuhaka-project-mapper:map-scan on YYYY-MM-DD.
> Enumeration source: <git ls-files | os.walk fallback>.
> ...

## Script-generated

| path | rule | decision | note |
| --- | --- | --- | --- |
| backend/ | MARKER | scatter | workspace marker: pyproject.toml |
| ... |

<!-- user-overrides below -->

| path/from/user/ | OVERRIDE | scatter | manual addition |
```

Two sections separated by the `<!-- user-overrides below -->` marker.

## Columns

| Column | Meaning | Allowed values |
|---|---|---|
| `path` | Directory path relative to project root, trailing slash | any relative path |
| `rule` | Which decision rule fired | `DENY`, `MARKER`, `BRANCH`, `LEAF`, `OVERRIDE` |
| `decision` | What map-sync does with this row | `scatter`, `skip` |
| `note` | Short reason | free text, one phrase |

`PASSTHROUGH` and `TRIVIAL` rules are NOT shown in the table (audit minimization — they have no map-sync effect).

## Script-generated vs user-overrides

The script regenerates the section above the `<!-- user-overrides below -->` marker on every `map-scan` run. The section below the marker is preserved verbatim.

User-overrides use cases:
- Force-scatter a directory the deny-list excluded (e.g., a `vendor/` fork you want documented)
- Add a directory the script missed (e.g., a sub-feature not yet detected)
- Force-skip a script-generated row (delete the row above and add an override below if needed)

Override rows should use `rule: OVERRIDE` for clarity.

## Example — Stocks repo

After running `/hukuhaka-project-mapper:map-scan` on a Stocks-shaped project:

```
| path | rule | decision | note |
| --- | --- | --- | --- |
| backend/ | MARKER | scatter | workspace marker: pyproject.toml |
| backend/app/ | BRANCH | scatter | 3 subdirs |
| backend/app/core/ | LEAF | scatter | 4 source files |
| backend/app/features/ | BRANCH | scatter | 5 subdirs |
| backend/app/features/canslim/ | LEAF | scatter | 6 source files |
| backend/app/features/macro/ | BRANCH | scatter | 2 subdirs |
| backend/app/features/macro/cycle/ | LEAF | scatter | 3 source files |
| ... | ... | ... | ... |
| frontend/ | MARKER | scatter | workspace marker: package.json |
| frontend/src/features/canslim/components/ | LEAF | scatter | 12 source files |
| ... | ... | ... | ... |
| vendor/canslim-quant-scanner/ | DENY | skip | deny-list: vendor |
```

## Idempotency

`map-scan` is deterministic over an unchanged tree. Running it twice produces an identical script-generated section. The script-generated section is fully regenerated each run; user-overrides section is appended unchanged.

## Editing scan.md by hand

Safe edits:
- Adding rows below `<!-- user-overrides below -->`
- Deleting your own rows below the marker

Unsafe edits (will be lost on next `map-scan`):
- Modifying script-generated rows above the marker
- Moving the marker
- Adding sections outside the documented layout

To permanently change script behavior (e.g., add a directory to deny-list), edit `scripts/scan.py` constants — not `scan.md`.
